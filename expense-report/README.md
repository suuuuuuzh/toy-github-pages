# 报销单网页

一个纯静态的报销单页面，用来给财务展示每笔报销的类目、金额、发票/T票凭证情况，并自动汇总发票总额、T票总额和总金额。不需要安装任何东西，本地双击 `index.html` 就能打开；以后推到 GitHub 后，会自动通过 GitHub Pages 发布，财务直接打开链接就能看。

## 目录结构

```
expense-report/
├── index.html      页面结构（一般不需要改）
├── style.css        样式（一般不需要改）
├── app.js           渲染逻辑，根据 data.js 自动计算汇总（一般不需要改）
├── data.js          ★ 每次报销要维护的数据
└── invoices/        存放发票 / T票扫描件的文件夹
```

## 什么是"替票"（T票）

有些费用没有正式发票，需要用别的发票/收据来抵，俗称"替票"或"T票"。替票和费用之间是**多对多**关系：
- 一张替票可以抵好几笔不同的费用（比如同一张收据抵了两笔市内打车）
- 一笔费用也可能要凑好几张不同的替票才够（比如一笔住宿费用两张替票合起来抵）

所以数据里把"替票"单独当成一份清单维护（每张票自己的编号、面值、扫描件），报销明细里的每一笔只需要写「这笔用了哪几张替票编号」。

## 每次报销怎么用

1. 把这次报销涉及的发票 PDF/图片、替票扫描件放进 `invoices/` 文件夹。
2. 打开 `data.js`，按下面的规则修改：
   - `reportInfo`：报销人、部门、报销周期、填报日期。
   - `expenseItems`：每一行是一笔报销，需要填：
     - `category` 类目、`description` 说明、`date` 日期、`amount` 金额
     - `voucherType`：
       - `"invoice"` —— 有正式发票，把发票文件路径填到 `invoiceFile`（例如 `"invoices/001-taxi.pdf"`）
       - `"tpiao"` —— 没有发票，用替票抵，把用到的替票编号填到 `tpiaoIds`（数组，可以填一个或多个，例如 `["T001"]` 或 `["T002", "T003"]`）
       - `"none"` —— 既没发票也没替票，页面会用红色高亮提醒需要补充凭证
   - `tpiaoList`：替票自己的清单，每张票填 `id`（编号）、`amount`（这张票本身的面值）、`file`（扫描件路径，可选）。同一个编号可以被多笔 `expenseItems` 引用。
3. 双击 `index.html` 在浏览器里预览，检查金额、汇总是否正确。
4. 确认无误后提交推送，GitHub Pages 会自动更新线上页面。

## 页面会自动算什么

- 顶部卡片：报销总金额、发票金额（及张数）、T票覆盖金额（走替票的业务金额，附带替票张数和面值合计），如果有无凭证的项目还会额外提示金额。
- 报销明细表：每行右侧可以直接点「下载发票」下载对应发票文件；走替票的行会列出所有关联的替票编号，点击可跳转到下方对应的替票汇总行。
- 替票汇总表：以替票为单位列出每张票自己的面值、关联了哪几笔费用（可能不止一笔），方便财务核对"一共用了几张替票、每张面值多少、都抵了哪些费用"。

## 之后如何正式对外托管

这个文件夹是纯静态页面，可以直接放在这个仓库里通过 GitHub Pages 访问（比如 `https://<你的用户名>.github.io/toy-github-pages/expense-report/`），也可以整体拷贝到公司自己的静态网站服务器上，不依赖任何后端。
