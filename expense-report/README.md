# 报销单网页

一个纯静态的报销单页面，用来给财务展示每笔报销的类目、金额、发票/替票凭证情况，并自动汇总发票总额、替票总额和总金额。不需要安装任何东西，本地双击 `index.html` 就能打开；以后推到 GitHub 后，会自动通过 GitHub Pages 发布，财务直接打开链接就能看。

## 目录结构

```
expense-report/
├── index.html      报销单页面结构（一般不需要改）
├── style.css        样式（一般不需要改）
├── app.js           渲染逻辑，根据 data.js 自动计算汇总（一般不需要改）
├── data.js          ★ 每次报销要维护的数据
├── invoices/        存放发票 / 替票扫描件的文件夹
├── import.html      "从 Excel 导入"工具页面（可选，帮你把 Excel 转成 data.js）
├── import.js         导入工具的解析 / 匹配逻辑
├── import.css        导入工具页面样式
└── lib/xlsx.full.min.js   解析 Excel 用的第三方库（SheetJS，本地文件，不依赖外网）
```

## 什么是"替票"

有些费用没有正式发票，需要用别的发票/收据来抵，就是"替票"（正式说法是"抵票"）。替票和费用之间是**多对多**关系：
- 一张替票可以抵好几笔不同的费用（比如同一张收据抵了两笔市内打车）
- 一笔费用也可能要凑好几张不同的替票才够（比如一笔住宿费用两张替票合起来抵）

所以数据里把"替票"单独当成一份清单维护（每张票自己的编号、面值、扫描件），报销明细里的每一笔只需要写「这笔用了哪几张替票编号」。

## 费用类目 / 发票类型

分类沿用了公司报销单模板的写法，两个字段是分开的：

- `category`（费用类目）：从 `categoryOptions` 里选，固定为 交通费 / 差旅-交通 / 住宿费 / 餐费 / 汽车费用 / 快递费 / 办公用品 / 业务招待费 / 其他。
- `invoiceCategory`（发票类型/发票内容）：发票上写的服务类别，例如"客运服务费""住宿费""餐票""过路费""停车费""油票""定额发票""核酸检测"等，和费用类目是两码事（比如"交通费"这个类目下的发票，发票类型可能是"客运服务费"）。走替票的项目，发票类型填在对应的 `tpiaoList` 条目上，页面会自动显示成"抵票-XXX"（例如"抵票-酒"）。

## 每次报销怎么用

1. 把这次报销涉及的发票 PDF/图片、替票扫描件放进 `invoices/` 文件夹。
2. 打开 `data.js`，按下面的规则修改：
   - `reportInfo`：报销人、部门、报销周期、填报日期。
   - `expenseItems`：每一行是一笔报销，需要填：
     - `category` 类目（从 `categoryOptions` 里选）、`description` 说明、`date` 日期、`amount` 金额
     - `voucherType`：
       - `"invoice"` —— 有正式发票，把发票文件路径填到 `invoiceFile`（例如 `"invoices/001-taxi.pdf"`），发票类型填到 `invoiceCategory`
       - `"tpiao"` —— 没有发票，用替票抵，把用到的替票编号填到 `tpiaoIds`（数组，可以填一个或多个，例如 `["T001"]` 或 `["T002", "T003"]`）
       - `"none"` —— 既没发票也没替票，页面会用红色高亮提醒需要补充凭证
   - `tpiaoList`：替票自己的清单，每张票填 `id`（编号）、`amount`（这张票本身的面值）、`invoiceCategory`（这张票的发票内容）、`file`（扫描件路径，可选）。同一个编号可以被多笔 `expenseItems` 引用。
3. 双击 `index.html` 在浏览器里预览，检查金额、汇总是否正确。
4. 确认无误后提交推送，GitHub Pages 会自动更新线上页面。

## 页面会自动算什么

- 顶部卡片：报销总金额、发票金额（及张数）、替票覆盖金额（走替票的业务金额，附带替票张数和面值合计），如果有无凭证的项目还会额外提示金额。
- 报销明细表：每行显示类目、发票类型、凭证类型；右侧可以直接点「下载发票」下载对应发票文件；走替票的行会列出所有关联的替票编号，点击可跳转到下方对应的替票汇总行。
- 费用类目汇总表：按 `category` 归集，列出每个类目一共几笔、金额合计，方便财务按类目核对总数。
- 替票汇总表：以替票为单位列出每张票自己的面值、发票类型、关联了哪几笔费用（可能不止一笔），方便财务核对"一共用了几张替票、每张面值多少、都抵了哪些费用"。

## 从 Excel 导入（可选）

如果你手上已经有按公司模板做好的报销 Excel（日期 / 费用摘要 / 费用明细按类目分列 / 发票内容 / 发票金额），
不用手动敲 `data.js`，打开 `import.html` 走一遍：

1. **第 1 步**上传报销 Excel —— 支持两种结构：
   - 公司模板（同时有"费用明细"和"发票内容"表头，费用按类目分列）；
   - 简单表格（表头有"金额"和"摘要/说明"即可，"日期""发票内容"可选）。
   没按类目分列的行，会根据摘要和发票内容里的关键词**自动划分类目**（机票/高铁→差旅-交通、
   打车/地铁→交通费、酒店→住宿费、招待→业务招待费、停车/油费→汽车费用、快递→快递费、
   办公/打印→办公用品、餐→餐费，其余归"其他"）。自动划分的在预览表格里显示为琥珀色下拉框，
   划错了直接改。一个 Excel 里有多个 sheet（比如每次出差一个 sheet）也会全部一起解析，
   同时会顺手把"报销人 / 部门 / 报销日期 / 项目名称"也读出来填进 `reportInfo`。
2. **第 2 步**选择这批发票文件（可多选）—— 只读取文件名，尝试从文件名里识别金额
   （比如文件名里带"141.00元"），不会读取文件内容，也不会上传到任何地方。
   金额能唯一对应上的会自动选好；金额重复、或者没找到匹配的，预览表格里会给一个下拉框手动选。
3. 核对预览表格没问题后，**第 3 步**复制或下载生成的 `data.js`，替换掉 `expense-report/data.js`；
   再把第 2 步里选中匹配上的发票文件（文件名不用改）放进 `expense-report/invoices/` 文件夹，提交推送。

这个工具只是把 Excel 转成 `data.js` 的一次性辅助，全过程都在浏览器本地完成。因为 Excel 表格本身
没有"发票文件名"这一栏，替票的"一票抵多笔 / 一笔用多票"这种关联，导入时每一行会默认生成自己的一张
替票，如果实际上有几笔费用共用同一张替票，导入后自己在 `data.js` 里把 `tpiaoIds` 改成同一个编号即可
（详见上面"什么是替票"的说明）。

## 之后如何正式对外托管

这个文件夹是纯静态页面，可以直接放在这个仓库里通过 GitHub Pages 访问（比如 `https://<你的用户名>.github.io/toy-github-pages/expense-report/`），也可以整体拷贝到公司自己的静态网站服务器上，不依赖任何后端。
